```js
_calculateNextDoseTime(treatment, lastScheduling, now) {
    const freqHours = treatment.frequency_hours || 0;
    if (freqHours <= 0) return null;
    const freqMs = freqHours * 60 * 60 * 1000;

    if (!treatment.start_date || !treatment.first_dose_time) return null;
    const [hours, minutes] = treatment.first_dose_time.split(':').map(Number);
    let baseTime = new Date(treatment.start_date);
    baseTime.setHours(hours, minutes, 0, 0);

    while (baseTime <= now) {
        baseTime = new Date(baseTime.getTime() + freqMs);
    }

    if (lastScheduling && lastScheduling.schedule_time) {
        const lastTime = new Date(lastScheduling.schedule_time);
        if (this._isSameDoseTime(lastTime, baseTime, freqMs)) {
            baseTime = new Date(baseTime.getTime() + freqMs);
        }
    }
    return baseTime;
},

_toLocalISOString(date) {
    const offset = date.getTimezoneOffset() * 60000;
    return new Date(date.getTime() - offset).toISOString().slice(0, 19);
}
```

A melhor forma depende do tamanho do projeto e de quantas pessoas trabalham nele. Para o **FarmaPocket**, eu recomendaria uma combinação simples:

## 1. Arquivo `TODO.md` na raiz do projeto
Ideal para controle rápido no dia a dia. Você abre, edita e acompanha o que falta.

Exemplo:
```markdown
# TODO FarmaPocket

## Em andamento
- [ ] Ajustar cálculo de próxima dose na Agenda de Hoje
- [ ] Testar fluxo de "Pular" dose

## Próximos passos
- [ ] Cadastro de sintomas
- [ ] Cadastro de eventos/procedimentos
- [ ] Cadastro de receituários

## Concluído
- [x] Login com Google
- [x] Cadastro de dependentes, medicamentos, tratamentos e profissionais
```

## 2. Issues no GitHub
Melhor para funcionalidades maiores, bugs e discussões. Cada item vira uma issue com descrição, labels e milestone.

Vantagens:
- Histórico completo
- Pode vincular a commits (`closes #12`)
- Fácil de priorizar

## 3. Comentários `TODO` no código
Bom para lembretes pequenos no próprio arquivo. Exemplo:
```javascript
// TODO: mover essa função para um módulo utilitário
// TODO: adicionar validação de dose máxima
```

Depois você pode listar todos com:
```bash
grep -r "TODO" js/
```

## Minha recomendação para você

Use **TODO.md** para o dia a dia + **Issues do GitHub** para funcionalidades e bugs maiores.
